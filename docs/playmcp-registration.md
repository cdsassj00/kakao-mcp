# 서버 1: 사람사전

**서버 이름**

```
사람사전
```

**한 줄 소개**

```
내 주변 사람들의 사전 — 약속, 취향, 기억을 사람 이름으로 찾아보세요.
```

**상세 설명**

```
"철수랑 마지막에 무슨 얘기했더라?", "지난달에 약속한 게 뭐였지?"
연락처에는 번호만 남고, 관계의 맥락은 사라집니다.

날씨나 뉴스는 AI가 이미 잘 압니다. AI가 절대 알 수 없는 단 하나가
당신의 관계입니다. 「사람사전」은 대화하다 "기억해줘" 한마디면
그 기억으로 '내 주변 사람들의 사전'을 만들어 줍니다.
사람 이름을 찾아보면 — 철수와 한 약속, 영희의 취향, 부장님과 나눈
이야기가 표제어처럼 펼쳐집니다.

이렇게 쓰세요:
1. "기억상자 만들어줘" — 나만의 사전과 열쇠(box_key)가 발급됩니다
2. "철수랑 다음주 화요일 점심 약속한 거 기억해줘" — 사람·종류·날짜와 함께 저장
3. "나 철수랑 뭐 하기로 했었지?" — 언제든 다시 찾아줍니다
4. "영희에 대해 내가 기억해둔 거 보여줘" — 취향/약속/메모를 사람별로 정리
5. "나 약속 뭐 있지?" — 약속만 날짜순으로

개인정보 원칙 — 내 열쇠로만 열리고, 언제든 완전히 지울 수 있습니다:
- 명시적으로 요청한 내용만 저장하며, 자동 수집이 없습니다
- 기억은 상자 단위로 격리되어 열쇠(box_key)를 가진 본인만 접근합니다
- 개별 삭제, 전체 영구 삭제, JSON 내보내기(백업)를 제공합니다
- 외부 API 호출·제3자 제공이 일절 없어 입력한 내용이 밖으로 나가지 않습니다

※ 열쇠를 잃어버리면 기억을 복구할 수 없으니 안전한 곳에 보관하세요.
```

**도구 요약**

```
- create_memory_box: 기억상자 생성과 접근 키 발급
- remember / recall: 기억 저장과 키워드·사람·종류 검색
- person_summary / list_people: 사람별 기억 요약과 인물 목록
- list_promises: 약속만 날짜순 조회
- forget / delete_memory_box / export_memories: 삭제·백업
- memory_stats: 보관 현황
```

**대화 예시**

```
기억상자 만들어줘
철수랑 다음주 화요일에 점심 먹기로 한 거 기억해줘
나 철수랑 뭐 하기로 했었지?
영희에 대해 내가 기억해둔 거 보여줘
나 약속 뭐 있지?
```

**태그 후보**: 인맥, 기억, 약속관리, 개인비서, 라이프스타일

**KC 빌드 설정**: Git URL `https://github.com/cdsassj00/kakao-mcp.git`, 브랜치 `main`, Dockerfile 경로 `Dockerfile`

# 서버 2: 말로 쓰는 가계부

**서버 이름**

```
말로 쓰는 가계부
```

**한 줄 소개**

```
"아아 4,500원" 한마디면 기록 끝 — 집계와 예산 경고는 서버가 정확하게.
```

**상세 설명**

```
가계부는 앱을 켜는 순간이 제일 어렵습니다. 그래서 대화가 곧 기록이
되게 만들었습니다.

"아아 4,500원", "어제 택시 만이천원" 처럼 말하면 AI가 금액과
카테고리를 알아듣고 기록합니다. 합계·카테고리 집계·예산 비교는
서버가 직접 계산하므로 AI의 암산 실수가 끼어들 수 없습니다.

이렇게 쓰세요:
1. "가계부 만들어줘" — 나만의 가계부와 열쇠(box_key)가 발급됩니다
2. "아아 4500원, 점심 12000원" — 여러 건도 한 번에 기록
3. "이번 달 얼마 썼어?" — 카테고리별 리포트와 예산 대비 현황
4. "식비 예산 40만원" — 이후 기록할 때마다 초과 여부 자동 경고
5. "이번 달 카페 내역 보여줘" — 조건별 내역 조회

개인정보 원칙:
- 기록은 열쇠(box_key)를 가진 본인만 접근합니다
- 계좌·카드 연동이 없습니다. 말한 것만 기록됩니다
- 내역 삭제, 전체 영구 삭제, JSON 내보내기(백업)를 제공합니다
- 외부 API 호출·제3자 제공이 일절 없습니다
```

**도구 요약**

```
- create_ledger: 가계부 생성과 접근 키 발급
- add_entries: 지출/수입 기록 (여러 건 일괄, 날짜 생략 시 오늘)
- monthly_report: 월간 카테고리별 리포트와 예산 대비
- list_entries: 월/카테고리별 내역 조회
- set_budget: 전체/카테고리 월 예산 설정 (80%·초과 자동 경고)
- delete_entry / delete_ledger / export_ledger: 삭제·백업
```

**대화 예시**

```
가계부 만들어줘
아아 4500원
어제 점심 12000원, 택시 8000원
이번 달 얼마 썼어?
식비 예산 40만원으로 잡아줘
```

**태그 후보**: 가계부, 소비관리, 예산, 절약, 라이프스타일

**KC 빌드 설정**: Git URL 동일, 브랜치 `main`, **Dockerfile 경로 `Dockerfile.ledger`**

## 경쟁 지형 메모 (2026-07-04, 등록 132개 기준)

- 대다수는 공공데이터/외부 API 래퍼 → "개인 데이터의 DB화"는 희소 포지션
- 사람사전 유사: **Heirmos(AI 영구 기억)** — 기술 중심 "메모리 인프라" 포지셔닝.
  우리는 '지식 기억'이 아닌 **'관계 기억'**(사람별 요약·약속·취향)으로 차별화
- 가계부: **132개 중 전무** — 완전 공백. 매일 쓸 이유가 있어 본선 투표에 유리
- 포화 영역(진입 금지): 모임 장소/일정/정산 (7개+), 경조사비(존재함)
- PlayMCP AI 채팅은 질문 수 제한(약 50개 + 수 분당 충전)이 있어
  긴 텍스트 붙여넣기형 UX는 심사 환경에서 불리 → 챗 완결 전략 채택 근거
