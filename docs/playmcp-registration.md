# 서버 1: 사람사전

## 등록 양식 카피

**서버 이름**

```
사람사전
```

**한 줄 소개**

```
내 주변 사람들의 사전 — 약속, 취향, 대화를 사람 이름으로 찾아보세요.
```

**상세 설명**

```
"철수랑 마지막에 무슨 얘기했더라?", "지난달에 약속한 게 뭐였지?"
연락처에는 번호만 남고, 관계의 맥락은 사라집니다.

날씨나 뉴스는 AI가 이미 잘 압니다. AI가 절대 알 수 없는 단 하나가
당신의 관계와 대화입니다. 「사람사전」은 외부 API 대신 사용자
동의 하에 그 기억으로 '내 주변 사람들의 사전'을 만듭니다.
사람 이름을 찾아보면 — 철수와 한 약속, 영희의 취향, 그때 그 식당이
표제어처럼 펼쳐집니다.

이렇게 쓰세요:
1. "기억상자 만들어줘" — 전용 기억상자와 열쇠(box_key)가 발급됩니다
2. "철수랑 다음주 화요일 점심 약속한 거 기억해줘" — 사람·종류·날짜와 함께 저장
3. "나 철수랑 뭐 하기로 했었지?" — 언제든 다시 찾아줍니다
4. "영희에 대해 내가 기억해둔 거 보여줘" — 취향/약속/메모를 사람별로 정리

과거 카카오톡 대화도 검색할 수 있습니다. 카카오톡 공식 '대화 내보내기'
텍스트를 붙여넣거나(짧은 대화), 챗이 건네주는 업로드 페이지에 파일을
올리면(긴 대화), "철수가 예약했다던 식당 이름 뭐였지?" 같은 질문에
대화 원문을 찾아 답합니다.

개인정보 원칙 — 내 열쇠로만 열리고, 언제든 완전히 지울 수 있습니다:
- 명시적으로 요청한 내용만 저장하며, 자동 수집이 없습니다
- 기억은 상자(box) 단위로 격리되어 열쇠(box_key)를 가진 본인만 접근합니다
- 개별 삭제, 채팅방 삭제, 전체 영구 삭제, JSON 내보내기(백업)를 제공합니다
- 외부 API 호출·제3자 제공이 일절 없어 입력한 내용이 밖으로 나가지 않습니다

※ 열쇠를 잃어버리면 기억을 복구할 수 없으니 안전한 곳에 보관하세요.
```

**도구 요약** (양식에 도구 설명란이 있는 경우)

```
- create_memory_box: 기억상자 생성과 접근 키 발급
- remember / recall: 기억 저장과 키워드·사람·종류 검색
- person_summary / list_people: 사람별 기억 요약과 인물 목록
- list_promises: 약속만 날짜순 조회
- import_kakao_export / upload_page_link: 카카오톡 대화 내보내기 가져오기
- search_chat / chat_context: 과거 대화 원문 검색과 앞뒤 맥락 조회
- forget / delete_chat_room / delete_memory_box / export_memories: 삭제·백업
- memory_stats: 보관 현황
```

**태그 후보**: 인맥, 기억, 대화검색, 약속관리, 개인비서, 라이프스타일

**Endpoint**: `https://<카카오클라우드 배포 주소>/mcp`
**KC 빌드 설정**: Git URL `https://github.com/cdsassj00/kakao-mcp.git`, 브랜치 `main`, Dockerfile 경로 `Dockerfile`

# 서버 2: 인맥 온톨로지

## 등록 양식 카피

**서버 이름**

```
인맥 온톨로지
```

**한 줄 소개**

```
내 카톡 대화가 관계망 지도가 됩니다 — 누구와 얼마나 가까운지 그래프와 강도로.
```

**상세 설명**

```
"나는 누구와 제일 자주, 제일 최근까지 이야기했을까?"

인맥 온톨로지는 카카오톡 공식 '대화 내보내기' 텍스트를 넣으면
관계망 그래프를 그려주는 MCP입니다. 사람을 노드로, 서로 주고받은
대화를 엣지로 연결하고, 대화 빈도 × 최근성 × 상호성을 결정론적으로
집계해 관계 강도(0~100)를 계산합니다. AI의 추측이 아니라 내 실제
대화 기록에서 나온 숫자입니다.

이렇게 쓰세요:
1. "관계망 만들어줘" — 나만의 관계망과 열쇠(box_key) 발급
2. 채팅방 ⚙️ 설정 → 대화 내보내기 → 텍스트 붙여넣기 (단톡방일수록 풍부)
3. "내 관계망 보여줘" — 가까운 사람 순위 + 관계망 지도(SVG) 링크
4. "나랑 철수 얼마나 친해?" — 상호작용 횟수·최근성 기반 상세 분석

개인정보 원칙:
- 사용자가 직접 넣은 대화만 분석하며, 자동 수집이 없습니다
- 관계망은 열쇠(box_key)를 가진 본인만 접근합니다
- 채팅방 단위 삭제와 전체 영구 삭제를 제공합니다
- 분석 결과는 조회 시 즉석 계산되며 제3자 제공이 없습니다
```

**태그 후보**: 인맥, 관계망, 소셜그래프, 데이터시각화, 자기분석

**Endpoint**: `https://<카카오클라우드 배포 주소 2>/mcp`
**KC 빌드 설정**: Git URL 동일, 브랜치 `main`, **Dockerfile 경로 `Dockerfile.ontology`**

## 경쟁 지형 메모 (2026-07-04, 등록 132개 기준)

- 대다수는 공공데이터/외부 API 래퍼 → "개인 데이터의 DB화"는 희소 포지션
- 직접 경쟁: **Heirmos(AI 영구 기억)** — 기술 중심 "메모리 인프라" 포지셔닝.
  우리는 '지식 기억'이 아닌 **'관계 기억'**(사람별 요약·약속·취향)으로 차별화,
  카톡 대화 원문 임포트·검색은 우리만 있음
- 선례: **밍글** — 카톡 '대화 내보내기' 기반 서비스가 이미 심사 통과 →
  임포트 방식의 심사 리스크 낮음을 확인
- 포화 영역(진입 금지): 모임 장소/일정/정산 (7개+), 경조사비(존재함)

## 심사 기준 대응 포인트

| 기준 | 우리 서비스 |
|---|---|
| 창의성 — 새로운 아이디어로 문제 해결·파급력 | 외부 API 래퍼가 아닌 "사용자의 사적 기록"을 AI 컨텍스트로 제공. 모든 카톡 사용자가 겪는 "대화는 흘러가고 남는 게 없다" 문제를 정면 공략 |
| 편의성 — 적합한 UI/UX, 일상의 실질 가치 | 챗 안에서 완결(저장·검색·요약·임포트). 서비스명만 봐도 용도가 이해됨. 긴 파일은 챗이 링크를 건네는 업로드 페이지로 해결 |
| 안정성 — 안정 구동·정확한 데이터·보안 | 외부 API 무의존이라 장애 요인 없음. 테스트 31건. 상자 격리·명시적 저장·삭제권(PRIVACY.md). 헬스체크 엔드포인트 제공 |
